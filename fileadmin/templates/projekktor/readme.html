UPGRADE INFORMATION:

read the changelog and take care if you are upgrading ...

from V0.8.21 and below

CSS Themes:

The start-button, the buffering indcator and the logo overlay now also use the new inactive/active CSS class system as described below. Please update your theme CSS accordingly. The offical themes have already been overhauled. You may want to use them as references.

YOUTUBE again:

The old Youtube flash API has been re-implemented and is now provided parallel to the iframed one. You can use the "useYTIframeAPI" config option to enable the iframe solution which is FALSE by default. Have fun!

from V0.8.17 and below

CSS Themes:

Projekktor removed all hardcoded "display"-CSS-directives from its JS code. With V0.8.18 the CSS-classes "active" or "inactive" are applied to the DOM elements to be toggled instead. As such you have to update your theme-CSS-files in order to work properly with this new principle. The offical themes have already been overhauled. You may want to use them as references.

YOUTUBE API:

Projekktor now supports the new, experimental and Flash-beating Youtube IFRAME API. Therefore Youtube videos will not work properly on IE6, IE7 and some other browsers not supporting "postMessage" and "JSON". Furthermore there are some known bugs in IE9 and Opera caused by Youtube´s javascripts.

Projekktor believes the benefit for the future is more valuable than the temporary loss. Anyhow, if your current project heavily depends on Youtube content we do not recommend to update right now.

Please keep in mind: Youtube support has been built into Projekktor for experimental reasons only. Projekktor believes that you should host your content for your own or use open hosters like BLIP.TV at least. Youtube is not the answer.
